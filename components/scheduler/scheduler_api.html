get_node_lock_by_id(node_id: int) → RWLock#

All read/write operations for a node should be locked within a per node rw lock that scheduler maintains. Usually you do NOT have to be concerned with this. But in cases you get the node object with functions like get_node_object_by_id. it is your responsibility to ensure data is locked when accessed. Lock is not part of the node itself.
